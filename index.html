程序的抽象语法的定义： \[ \begin{aligned} S ::=&\; \mathsf{x = y} \\ | &\; \mathsf{x = \&y} \\ | &\; \mathsf{x = \ast{}y} \\ | &\; \mathsf{x = op(y_1 \ldots y_n)} \\ | &\; \mathsf{x = allocate(y)} \\ | &\; \mathsf{\ast{}x = y} \\ | &\; \mathsf{x = fun(f_1 \ldots f_n) \to (r_1 \ldots r_m)}\, S^\ast \\ | &\; \mathsf{x_1 \ldots x_m = p(y_1 \ldots y_n)} \end{aligned} \]

类型的定义： \[ \begin{aligned} &\alpha \; ::= \; \tau \times \lambda \\ &\tau \; ::= \; \bot \,|\, \mathbf{ref}(\alpha) \\ &\lambda \; ::= \; \bot \,|\, \mathbf{lam}(\alpha_1 \ldots \alpha_n) (\alpha_{n+1} \ldots \alpha_{n+m}) \end{aligned} \] 其中 \(\alpha\)表示值，\(\tau\) 表示地址以及指向地址的指针。\(\lambda\) 表示函数以及指向函数的指针。

Maryam Emami 1994

先看下这篇文章 Context-sensitive interprocedural points-to analysis in the presence of function pointers, PLDI 94, link

Defintion abstract stack location xat program point \(p\) represents one or more real stack locations.

real stack locations can be (1) the name of a local variable, global variable or parameter; or (2) a syumbolic name that corresponds to locations indirectly accessible through a parameter or global variable(of pointer type), when these locations correspoind to variables not in the scope of the procedure under analysis; or (3) the symbolic name heap

Definition \(RSL\) : all real stack locations.

Definition \(ASL\) : all abstract stack locations.

Definition \(\ast(x) : ASL \mapsto 2^{RSL}\), the map from \(ASL\) to the power set of \(RSL\),

Definition points-to relation \((x, y, D)\) if \(|\ast(x)| = |\ast(y)| = 1\) and

ASL x y, if x and y each represent exactly one real stack location in a particular invocation context, and the real stack location correspoinding to x contains the address of the real stack location correspoinding to y

Definition definitely points-to relation

用图集合，表示程序点变量的所有可能形状。

目标语言定义：

\[ \begin{aligned} program & \to \{[label:] stmt\}^\ast \\ stmt & \to assign \,|\, \mathit{if} \,|\; goto \\ assign & \to var := exp \,|\, var := \mathsf{input} \,|\, \mathsf{output} := exp \\ \mathit{if} & \to \textsf{if} \, test \, goto \\ test & \to \mathsf{atom} \; exp \,|\, \mathsf{null} \; exp \,|\, var \, \{= \,|\, \neq\} \, var \\ goto & \to \mathsf{goto} \; label \\ exp & \to atom \,|\, var \,|\, var.sel \,|\, \mathsf{cons}(exp \; \{, exp\}^*) \end{aligned} \]

其中 \(var\), \(sel\), \(atom\), \(label\) 分别是互不相交的集合 \(Var\), \(Sel\), \(Atom\), \(Label\) 中的元素。本文中 \(Sel = \{ \mathsf{hd}, \mathsf{tl}\}\) 。

函数 \(\mathfrak{A} : Atom \to \mathit{ATOM}\) 是原子到值的映射，值可以是 数字，有限长度的字符串，布尔值等。

Definition: Label sequence 定义为 \(s_1.s_2. \ldots .s_n . a\), where \(n \geq 0\), and \(s_1, \ldots s_n \in Sel\) , and \(a \in ATOM\) , 即 label sequences 就是 \(Sel^\ast \times ATOM\)

Definition：Tree , Label sequence 的有限子集。空集的树写作 \(\bot\) .

用这种集合来表示树的例子：

\(\bot\) : \(\emptyset\)

1 : \(\{1 \}\)

: \(\{ \mathsf{hd}.1 , \mathsf{tl}.2\}\)

: \(\{ \mathsf{tl.hd.hd}.2 , \mathsf{tl.tl}.3 \}\)

如果 \(T, T_1, \ldots, T_m\) 是树， \(s \in Sel\) , 求子树的操作为 \(T.s = \{ t_1\ldots t_n.atom \,|\, s.t_1 \ldots t_n.atom \in T \}\); 合成多个树的操作为 \(\mathsf{cons}(T_1,\ldots,T_m) = \cup^m_{i=1} \{ sel_i.t_1 \ldots t_n.atom \,|\, t_1 \ldots t_n.atom \in T_i \}\)

\(\mathit{VAL} = 2^{Sel^* \times ATOM}\) , 所有的树的集合，也是该语言中的值。

\(\mathit{STORE} = Var \to \mathit{VAL}\) ，变量名到值的所有映射的集合。一个 store 实例 \(\sigma \in \mathit{STORE}\) 把每个变量映射到它们的当前值。

指称语义 这里只定义了表达式求职和赋值语句的执行。详见它的引文了。

\(\mathfrak{E} : Exp \to \mathit{STORE} \to \mathit{VAL}\) 是如下定义的 partial function:

\(\mathfrak{E}[[atom]] \sigma = \{ \mathfrak{A}[[ atom ]] \}\) .

\(\mathfrak{E}[[var]] \sigma = \sigma(var)\)

\(\mathfrak{E}[[var.s]] \sigma = \sigma(var.s) \; ,\text{undefined if } \sigma(var) = \bot \text{ or is an atom}\)

\(\mathfrak{E}[[\mathsf{cons}(e_1,\ldots, e_m)]] \sigma = \mathsf{cons}(\mathfrak{E}[[e_1]]\sigma,\ldots,\mathfrak{E}[[e_m]]\sigma)\)

\(\mathfrak{AS} : Assign \to \mathit{STORE} \to \mathit{STORE}\) 定义如下：

\(\mathfrak{AS}[[var := exp]] \sigma = \lambda x \in Var. (\mathsf{if}\; x = var \; \mathsf{then}\; \mathfrak{E}[[exp]]\sigma \; \mathsf{else} \; \sigma(x))\)

Structure Shapes and Data Flow Equations

定义数据流分析。

\(X \in Var\) 是程序变量，\(I\) 是程序点，变量 \(F(I,X)\) 表示程序点 \(I\) 处变量 \(X\) 取值（形状）的集合。不动点算法的解最终得到的 \(F(I,X)\) 中是所有可能执行路径下，程序点 \(I\) ，变量\(X\) 的所有可能形状的集合。

形状的定义 \(Shape = 2^{Sel^* \times \{o\}}​\) ，对比上面定义的 \(Val​\) , 这里把所有的 \(\mathit{ATOM}​\) 都用 \(o\) 来表示。格为 \(2^{Shape}\)，偏序关系为子集。即 \(F(I,X) \in 2^{Shape}\).

先把 SL 程序转换成流图 (算法见引文)，标记 program point 集合\(\mathfrak{PP} = \{0,1,\ldots,n\}\). 流图的边都是一个程序点。

流图中，非-if 语句 出度为1，if语句 出度为 2。记语句的入边为 \(I_1, \ldots, I_n\), 语句的出边为 \(J\) （ 或 \(J_Y, J_N\) ，对于 if 语句）， 数据流方程如下：

  • \(X := \mathsf{atom} \text{ or } X := \mathsf{input}\)
  • \(F(J,X) = \{o\}\)
  • \(X := Y\)
  • \(F(J,X) = \cup_{i=1}^p F(I_i, Y)\)
  • \(X := Y.s\) \(F(J,X) = \cup_{i=1}^p F(I_i,Y).s\)
  • \(X := \mathsf{cons}(Y_1,\ldots,Y_m)\)
  • \(F(J,X) = \cup_{i=1}^p \mathsf(cons)(F(I_i, Y1),\ldots, F(I_i,Y_m))\)
  • \(\mathsf{if\;atom}\;X\)
  • \(F(J_Y,X) = \cup_{i=1}^p F(I_i,X) \cap \{o\}\)
  • \(F(J_N,X) = \cup_{i=1}^p F(I_i,X) - \{o\}\)
  • \(\mathsf{if\;null}\;X\)
  • \(F(J_Y,X) = \cup_{i=1}^p F(I_i,X) \cap \{\bot\}\)
  • \(F(J_N,X) = \cup_{i=1}^p F(I_i,X) - \{\bot\}\)
  • \(\mathsf{if\;}X = Y\)
  • \(F(J_Y,X) = \cup_{i=1}^p (F(I_i,X) \cap F(I_i,Y)) \cap \{o,\bot\}\)
  • \(F(J_N,X) = \cup_{i=1}^p F(I_i,X) \cap \{o,\bot\}\)
  • \(F(J_Y,Z) = F(J_N,Z) = \cup_{i=1}^p F(I_i,Z) \text{ for }Z \neq X,Y\)
  • \(\text{other}\)
  • \(F(J,X) = \cup_{i=1}^p F(I_i,X)\)

初始值 \(F(I_0,X) = \{\bot\}\)

但是这个格并不是有限的，因此接下来定义一个 Tree Grammars 来近似 Shape。
